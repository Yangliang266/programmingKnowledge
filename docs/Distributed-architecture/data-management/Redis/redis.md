## 高并发

### 1 缓存雪崩

1. random
2. 永不过期
3. 预更新

### 2缓存穿透

1. 现象：数据查询由于key的不存在，但又满足查询条件，会绕过redis进行数据库查询
2. 问题：数据量巨大进行查询，数据库负载较大
3. 解决：布隆过滤器，判断是否存在，保证redis的使用
   1. 原理：利用哈希表映射到位图的特性（通过多次hash计算映射到位图）
   2. 缺点：当大数据量过滤时，会有hash值的碰撞，有一定的误判率
